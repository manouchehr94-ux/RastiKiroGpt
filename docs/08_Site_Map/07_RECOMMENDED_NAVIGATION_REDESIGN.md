## ۲. پیشنهاد bottom navigation تکنسین (موبایل)

### ساختار فعلی
```
[داشبورد] [سفارش جدید] [سفارش‌های من] [فاکتورها] [اعلان‌ها]
```

این ساختار مناسب است. پیشنهاد می‌شود:

1. **آیکون اعلان با badge** — نمایش تعداد اعلان‌های خوانده‌نشده
2. **وضعیت فعال** — آیتم فعال با رنگ متفاوت (در حال حاضر به جای text رنگ `brand-600` استفاده می‌شود — این درست است)
3. **desktop view** — برای desktop باید sidebar معادل نشان داده شود

## ۵. پیشنهاد ناوبری مالی ادمین

**مشکل فعلی:** گزارش‌های مالی (۶ صفحه) و عملیات پرداخت (۳ صفحه) در sidebar ادمین بدون ساختار هستند.

**پیشنهاد:** یک صفحه "مرکز مالی" (`/<code>/admin/financial/`) که به عنوان hub تمام صفحات مالی عمل کند:

```
مرکز مالی
├── گزارش خلاصه
├── تفکیک تکنسین‌ها
├── تسویه فاکتورها
├── کنترل نقدی
├── کارمزد پلتفرم
├── گزارش حسابرسی
├── عملیات پرداخت
├── تسهیم پرداخت‌ها
└── مغایرت‌گیری درگاه
```

## ۶. پیشنهاد ناوبری تنظیمات (Settings)

**مشکل فعلی:** تنظیمات پراکنده هستند — برخی در `/admin/settings/`، برخی در `/admin/sms/`، برخی در `/admin/communication-settings/`.

**پیشنهاد:** یک صفحه "مرکز تنظیمات" که از `includes/settings_center.html` الهام گرفته:

```
تنظیمات شرکت
├── اطلاعات پایه (admin/settings/)
├── برندینگ (admin/branding/)
├── صفحه عمومی (admin/page/)
├── فیلدهای سفارشی (admin/custom-fields/)
├── اپراتورها (admin/settings/operators/)
├── پیامک
│   ├── قالب‌ها (admin/sms/templates/)
│   ├── تنظیمات اعلان (admin/settings/notifications/)
│   ├── ارتباطات (admin/communication-settings/)
│   └── تشخیص (admin/sms/diagnostics/)
└── پرداخت
    ├── درگاه (admin/payment-gateway/)
    └── پروفایل KYC (admin/payment/merchant-profile/)
```

## ۷. پیشنهاد ناوبری پیامک/اعلان ادمین

**مشکل فعلی:** سه بخش جداگانه برای پیامک:
1. `/<code>/admin/sms/` — outbox، templates، inbox
2. `/<code>/admin/sms-credit/` — کیف پول
3. `/<code>/admin/communication-settings/` — تنظیمات رویداد

**پیشنهاد:** یک breadcrumb منطقی:
```
پیامک و ارتباطات
├── صندوق خروجی
│   └── /<code>/admin/sms/outbox/
├── صندوق دریافت
│   └── /<code>/admin/sms/inbox/
├── قالب‌های پیامکی
│   └── /<code>/admin/sms/templates/
├── تشخیص و تست
│   └── /<code>/admin/sms/diagnostics/
├── کیف پول پیامک
│   └── /<code>/admin/sms-credit/
└── تنظیمات اعلان
    ├── /<code>/admin/settings/notifications/
    └── /<code>/admin/communication-settings/
```

## ۹. پیشنهاد breadcrumb استاندارد

باید به هر صفحه داخلی breadcrumb اضافه شود. الگوی پیشنهادی:

```
داشبورد > تکنسین‌ها > [نام تکنسین] > صورتحساب
داشبورد > سفارشات > #1234 > اختصاص تکنسین
گزارش مالی > تفکیک تکنسین
```

قالب موجود `components/breadcrumb.html` باید در `base_dashboard.html` با یک `{% block breadcrumb %}` فعال شود.
