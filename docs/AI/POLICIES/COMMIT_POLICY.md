# Commit Policy

One request must produce one focused commit.

Do not mix unrelated work.

---

## Commit Types

- feat(scope): new feature
- fix(scope): bug fix
- refactor(scope): internal restructuring
- docs(scope): documentation
- test(scope): tests
- chore(scope): maintenance

---

## Examples

fix(payments): render public checkout without dashboard layout

fix(invoices): block technician invoice after cancellation request

docs(ai): add workspace development protocol
