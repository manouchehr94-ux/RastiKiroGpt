# Project Principles

1. Correctness before speed.
2. Small safe changes before broad rewrites.
3. Service layer before view logic.
4. Tests before risky changes.
5. Tenant isolation everywhere.
6. Money logic must be deterministic and auditable.
7. No hidden side effects in financial code.
8. Every important decision must be documented.
9. Production data must be protected.
10. If unsure, stop and report.
